班级链接：百校未来星-中原工学院-（HarmonyOS应用开发-2025秋季学期）-华为开发者学堂

鸿蒙开发学习笔记：从基础到实战的进阶指南

一、学习规划与准备

（一）核心学习原则

遵循 “聚焦核心、实战导向、高频回顾” 三原则，优先掌握 20% 高频知识点解决 80% 问题，每个知识点配合案例练习，并坚持每日回顾巩固记忆。建议每天投入 4-6 小时，采用 “45 分钟学习 + 10 分钟休息” 的节奏保持效率。

（二）开发环境搭建

  1. 工具安装：从华为开发者官网下载 DevEco Studio，安装时选择 “HarmonyOS 开发”，自动配置 Node.js 环境并安装最新稳定版 HarmonyOS SDK。

  2. 环境配置：启动后配置模拟器（推荐 P50 Pro 机型），注册华为开发者账号获取远程模拟器权限，确保 JDK 8 以上及 Gradle 工具正常运行。

  3. 项目初始化：点击 “Create Project” 选择 “Empty Ability” 模板，填写项目信息（如 Project Name: MyFirstApp），等待依赖加载完成后运行验证环境。

二、核心基础技术

（一）ArkTS 语言核心

  1. 数据类型与变量：支持 number、string、boolean 等基本类型，用 let 声明可变变量，const 声明不可变变量，需显式添加类型注解。

  1. 函数与控制流：函数需指定参数和返回值类型，支持 if-else、for、switch 等控制语句，语法与 TypeScript 兼容。

  2. 状态管理：通过@State装饰器实现数据驱动 UI 更新，@Builder封装复用 UI 组件，需遵循 build () 函数根节点唯一的规范。

（二）核心开发概念

  1. 应用结构：以 Ability 为基本单元，分为带 UI 的 Page Ability 和后台服务的 Service Ability，项目通过 HAP（应用包）和 HSP（共享包）实现弹性部署。

  2. 声明式 UI：采用 UI 描述与逻辑分离模式，通过 Column、Row 等容器组件和 Text、Button 等基础组件构建界面，状态变化自动触发 UI 更新。

  3. 资源管理：静态资源存放于resources/base/media目录，通过$r(‘app.media.xxx‘)方式引用。

三、UI 开发与实战案例

（一）常用组件与布局

  1. 基础组件：

  • Text：控制字体大小（fontSize）、粗细（fontWeight）等样式

  • Button：通过 onClick () 绑定点击事件，设置背景色与尺寸

  • TextInput：用 placeholder 设置提示文本，onChange 获取输入内容

  1. 布局容器：

  • Column/Row：实现垂直 / 水平线性布局，通过 justifyContent 控制对齐方式

  • Stack：层叠布局，用于叠加显示图片与文字

  • List：通过 ForEach 循环渲染列表数据

（二）个人名片应用实现

运行后在模拟器中调试布局，通过调整 margin 和 spacing 优化界面间距。

四、进阶技巧与避坑指南

（一）分层架构设计

采用 “产品定制层→基础特性层→公共能力层” 三级结构：

  • 产品定制层（entry）：按设备（手机 / 平板）实现专属 UI

  • 基础特性层（features）：封装新闻、设置等独立业务模块

  • 公共能力层（common）：提供网络请求、工具类等复用能力
